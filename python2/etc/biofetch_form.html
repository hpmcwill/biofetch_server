Biofetch Meta-data

  • Maximum number of identifiers for a request: biofetch.cgi?info=maxids
  • List of databases: biofetch.cgi?info=dbs
  • Data formats for a database: biofetch.cgi?info=formats;db=embl
  • Result styles for a data format of a database: biofetch.cgi?info=styles;db=embl;format=fasta

Fetch Data

  • Default database: biofetch.cgi?id=L12345
  • EMBL-Bank:
    • L12344 and L12345 in EMBL format: biofetch.cgi?db=embl;format=html;style=html;id=L12344,L12345
    • L12344 and L12345 in fasta format: biofetch.cgi?db=embl;format=fasta;style=raw;id=L12344,L12345
  • UniProtKB:
    • WAP_MOUSE, WAP_PIG and WAP_RAT in UniProt flat-file format: biofetch.cgi?db=uniprotkb;id=WAP_MOUSE,WAP_PIG,WAP_RAT
    • P01173, O46655 and P01174 in fasta format: biofetch.cgi?db=uniprotkb;format=fasta;style=raw;id=P01173,O46655,P01174

Error Messages

  • Error 1: Unknown database.
  • Error 2: Unknown style.
  • Error 3: Unknown format.
  • Error 4: ID not found.[1]
  • Error 5: Too many identifiers.
  • Error 6: Unknown information.
  • Error 11: Unable to connect to database.
  • Error 12: No entries found.
  • Error 0: Unknown error.

1. The use of batch queries for many data sources means that identifiers which occur in the request but not in the result cannot easily be identified. Thus "Error 4" is typically not used and a "no entries found" error ("Error 12") is used instead.
